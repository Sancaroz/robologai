Robot Asset Intake

Turn uploaded robot images into modular data updates.

1. Folder shape assets/robots/company-slug/robot-slug/hero.jpg Add gallery-1.jpg, gallery-2.jpg, and source.txt in the same folder when available.
2. Preview import node scripts/import-robot-assets.mjs --company company-slug Dry-run first; it shows which modular robot JSON files will be created or updated.
3. Write records node scripts/import-robot-assets.mjs --company company-slug --write Updates image, heroImage, gallery, aggregate JSON, generated profiles, and validation checks.
4. Check image quality node scripts/audit-robot-images.mjs --company company-slug --min-hero 900 --min-card 600 Flags local robot images that are card-ready, hero-ready, too small, or unreadable.
5. Draft safety source.txt or --create-drafts New robot records are created only with a source URL, unless draft creation is explicitly enabled.
6. Final gate node scripts/health-check.mjs Use data-ops, company pages and robot pages before commit.

Data Quality Queue

Work from the highest-risk records first.

LoadingBuilding priority queue...Assets, source links, profile pages, and commercial clarity are being ranked into an action list.
Robot candidate discovery node scripts/discover-robot-candidates.mjs --input leads.txt --write Turns batch URL lists into ignored local candidate reports before modular robot records are created.
Robot candidate review node scripts/review-robot-candidates.mjs --markdown Compares candidate reports against the catalog and prints add/update/duplicate review actions.
Source audit node scripts/audit-sources.mjs --type robot --limit 20 Use after adding product pages or official references so robot and company records move toward two independent source URLs.
Company source audit node scripts/audit-sources.mjs --type company --limit 20 Lists modular company records that need deeper website, product, press, docs, or investor sourceLinks.
Stricter review node scripts/audit-sources.mjs --company company-slug --min 3 Useful for important companies before publishing a larger batch of robots or market signals.
Price signal preview node scripts/watch-price-signals.mjs --company company-slug --limit 10 Shows which official robot source URLs would be scanned before any network request is made.
Fetch price signals node scripts/watch-price-signals.mjs --fetch --company company-slug --limit 10 Looks for price, deposit, preorder, quote, and contact-sales signals without changing robot cards.
Review report node scripts/watch-price-signals.mjs --fetch --write --limit 50 Saves ignored local JSON reports under data/price-signals/ for manual review before publishing any price.
Review candidates node scripts/review-price-signals.mjs --min-confidence 4 Compares saved signals with existing structured prices and prints review actions without editing data.
Draft price JSON node scripts/review-price-signals.mjs --drafts Prints clean structured price draft records for manual copy into data/prices after review.
Markdown review node scripts/review-price-signals.mjs --markdown Prints a checkbox review table for comparing current structured prices with new candidate signals.
